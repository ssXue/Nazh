# ADR-0022：工作流变量持久化

- **状态**：已接受
- **日期**：2026-05-03
- **决策者**：ssXue
- **关联**：ADR-0012 Phase 3、RFC-0003 Phase 1

## 背景

变量当前纯内存（`DashMap`），进程退出即清零。工业边缘场景需要 setpoint、计数器等在重启后恢复。RFC-0003 已提议 `crates/store/` SQLite 共享 crate，本 ADR 实施其 Phase 1 子集——仅覆盖变量持久化、历史记录和全局变量表。

## 决策

> 我们决定新建 `crates/store/` Ring 1 crate（SQLite），实现 RFC-0003 的最小集：变量写入时同步持久化，部署时从 Store 恢复。

### 表结构

```sql
-- 变量快照（每个 workflow_id + key 唯一）
CREATE TABLE variables (
    workflow_id TEXT NOT NULL,
    key         TEXT NOT NULL,
    value       TEXT NOT NULL,        -- JSON serialized Value
    var_type    TEXT NOT NULL,        -- PinType 序列化
    initial     TEXT NOT NULL,        -- JSON serialized initial Value
    updated_at  TEXT NOT NULL,
    updated_by  TEXT,
    PRIMARY KEY (workflow_id, key)
);

-- 变更历史（追加写）
CREATE TABLE variable_history (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    workflow_id TEXT NOT NULL,
    key         TEXT NOT NULL,
    value       TEXT NOT NULL,
    updated_at  TEXT NOT NULL,
    updated_by  TEXT
);
CREATE INDEX idx_var_history_time ON variable_history(workflow_id, key, updated_at);

-- 全局变量（跨工作流共享）
CREATE TABLE global_variables (
    namespace  TEXT NOT NULL,
    key        TEXT NOT NULL,
    value      TEXT NOT NULL,
    var_type   TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    updated_by TEXT,
    PRIMARY KEY (namespace, key)
);

-- Schema 版本管理
CREATE TABLE schema_version (
    version    INTEGER PRIMARY KEY,
    applied_at TEXT NOT NULL
);
```

## 可选方案

### 方案 A：JSON 文件（如 connections.json 模式）

优势：无新依赖，实现简单。
劣势：无事务语义、无查询能力、文件膨胀后性能差、无法支持历史查询。

### 方案 B：SQLite 共享 crate（选择）

优势：RFC-0003 已提议，`rusqlite` 已在 workspace（`nodes-io` 已验证 bundled 编译），desktop/daemon 共用，支持事务和复杂查询。
劣势：新增 crate 维护、SQLite bundled ~5MB 产物。

### 方案 C：JSONL append-only（如 observability 模式）

优势：追加写性能高。
劣势：无事务、无查询能力、文件膨胀。

### 方案 D：延迟到 edge-daemon 时实施

优势：减少当前工作量。
劣势：变量持久化、审计追溯能力继续缺失。

## 后果

### 正面影响

- 变量在进程重启后可恢复
- 历史曲线功能获得数据基础
- 全局变量实现跨工作流状态共享
- 为 RFC-0003 后续 Phase（可观测事件索引、审计）奠定基础

### 负面影响

- 新增 `crates/store/` crate 维护
- SQLite bundled 编译 ~5MB 产物增大
- 变量写入延迟增加（同步 SQLite UPSERT，通常 < 1ms）

### 风险

- **交叉编译 SQLite**：`nodes-io` 已在 x86_64 macOS 验证 bundled 编译，ARM 边缘设备需额外验证。缓解：CI 加入 ARM target check。
- **恢复时序**：变量恢复在 `build_workflow_variables` 之后、节点 `on_deploy` 之前。节点 `on_deploy` 读到的变量值包含恢复值——语义正确。
- **Store 与 Ring 0 耦合**：Store 直接消费 `serde_json::Value` 和 `String` 类型做序列化，不依赖 Ring 0 类型。Ring 0 保持纯净。

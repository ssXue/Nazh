# RFC（Request for Comments）

本目录存放 Nazh 项目的功能提案。

## 什么是 RFC？

RFC 是在实现一个较大的功能或架构变更**之前**，先写出设计文档供讨论的流程。
与 ADR 的区别：

| | ADR | RFC |
|---|---|---|
| 时机 | 决策**之后**记录 | 决策**之前**提出讨论 |
| 体量 | 轻量（1-2 页） | 可以较长（含技术方案细节） |
| 目的 | 记录"为什么这样做" | 讨论"应该怎么做" |

## 如何提交 RFC

1. 复制 `template.md` 为 `NNNN-简短标题.md`（编号递增）
2. 填写各个章节
3. 提交 PR，在 PR 中讨论
4. RFC 被接受后，状态改为"已接受"，进入实施阶段
